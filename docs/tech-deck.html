Payments

x402 Payment Flow

  1. 01 Agent sends HTTP request with x402 payment header
  2. 02 Coordinator validates payment and extracts task
  3. 03 Task routed to optimal worker based on policy
  4. 04 Worker executes task and generates signed receipt
  5. 05 Coordinator verifies receipt and anchors on-chain
  6. 06 Result returned to agent with cryptographic proof
Verification

Trust Without Intermediaries

SIGNATURES

Ed25519

Every worker result is cryptographically signed. Agents can independently verify without trusting the coordinator.

IDENTITY

ERC-8004

On-chain agent identity and reputation. Workers build verifiable track records across tasks.

PROOF

On-Chain Anchoring

Execution receipts anchored to Solana or Monad. Immutable, auditable, permanent.

Routing

Smart Task Routing

FAST

Speed Priority

Route to nearest available worker. Minimize latency. Best for real-time agent decisions.

CHEAP

Cost Priority

Route to lowest-cost worker. Batch when possible. Best for background processing.

PRIVATE

Privacy Priority

Route to TEE-enabled workers. Encrypted execution. Best for sensitive data.
